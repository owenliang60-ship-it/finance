**日期:** 2026-04-24

**范围:** 在原 RVOL 事件统计基础上，加入 PMARP 状态分桶；不是策略回测

## 一句话结论

这轮只改一个东西：把 `RVOL_150 cross_up 2σ` 事件按信号日 PMARP 数值分成三类：

- `pmarp_lt30`: `PMARP < 30`
- `pmarp_30_60`: `30 <= PMARP < 60`
- `pmarp_gte60`: `PMARP >= 60`

其他统计口径全部不变。

核心结论：

1. `RVOL_150 cross_up 2σ` 在 `pool` 和 `extended` 两个股票池上，后续原始收益仍然整体为正。
2. `PMARP 30-60` 是最弱的中间状态桶，在两个股票池的相对 SPY 超额收益口径下都没有通过显著性检验。
3. `pool` 里，`PMARP < 30` 和 `PMARP >= 60` 都有超额收益 edge，但 `PMARP >= 60` 的均值 alpha 更干净。
4. `extended` 里，只有 `PMARP >= 60` 在 `20d` 超额收益上通过 FDR；`40d` 变弱。
5. 当前更合理的解释是：RVOL 不是纯粹的 panic bottom 信号。更干净的横截面 alpha 来自“强势状态里的放量确认”，而不是“所有恐慌放量都买”。

## 固定统计口径

### 信号定义

- `RVOL_150 cross_up 2σ` = `prev_rvol <= 2.0 and curr_rvol > 2.0`

### 收益口径

- 原始 close-to-close 收益: `close[T+h] / close[T] - 1`
- 相对 SPY 超额收益: 个股原始收益减去 SPY 原始收益
- Horizons: `5 / 10 / 20 / 40`

### 统计纪律

- 同标的持有期去重：同一只股票在前瞻持有期内重复触发，只保留第一个事件
- 日期聚类 t 检验：同一天多个股票触发，先按日期取横截面均值，再做 t-test
- BH-FDR：按 `universe + return_type` 家族校正

## 原始收益一致性校验

原始收益维度下，各桶仍然整体为正：

| 股票池 | 分桶 | 20d 原始收益均值 | 40d 原始收益均值 |
|---|---|---:|---:|
| `pool` | `PMARP < 30` | +2.99% | +4.97% |
| `pool` | `PMARP 30-60` | +2.25% | +3.84% |
| `pool` | `PMARP >= 60` | +2.95% | +4.45% |
| `extended` | `PMARP < 30` | +2.05% | +3.59% |
| `extended` | `PMARP 30-60` | +1.45% | +2.91% |
| `extended` | `PMARP >= 60` | +1.36% | +2.10% |

这个一致性校验的意义是：低 PMARP 的反弹不是假的，但很大一部分可能是市场 beta / 大盘反弹；一旦换成相对 SPY 超额收益，就没有广义股票池 alpha 证据。

## 当前解释

这一轮之后，不应该简单说“低 PMARP 的 RVOL 最好”。更贴近数据的解释是：

- `PMARP >= 60`: RVOL 更像确认/加速信号，尤其在精选 `pool` 里 alpha 更干净。
- `PMARP < 30`: RVOL 可能抓到反弹，但广义股票池里的超额收益 alpha 弱，需要额外过滤器。
- `PMARP 30-60`: 暂时不要作为高优先级 RVOL 事件类。

下一步更值得问的问题是：

> RVOL 是否应该作为强势股的确认信号使用，而低 PMARP 的 RVOL 需要额外过滤器后才可交易？
