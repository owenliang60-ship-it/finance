# 热点追踪系统 (Trend Tracker) — 设计蓝图

> **层级**: 分析层 (Terminal)
> **日期**: 2026-03-20
> **状态**: 设计讨论完成，待实现

---

## 一、核心命题

**如何在市场趋势形成的早期（阶段②点火）就机械地检测到它，而不依赖人的注意力带宽？**

### 问题拆解

一个人管理百只以上的股票池，注意力是不可扩展的资源。等某个主题"穿透"人的注意力阈值时，趋势往往已经进入阶段③（扩散），错过了前 15-50% 的涨幅。

### 趋势形成的五个阶段

| 阶段 | 信号特征 | 粗估涨幅 |
|------|---------|---------|
| ① 暗流 | 少数人在讨论，价格还没动 | 0% |
| ② 点火 | 催化剂出现，龙头先动，量放大 | 0→15% |
| ③ 扩散 | 分析师跟进，板块内多股联动，社交热度上升 | 15→50% |
| ④ 主升浪 | 人人都知道，资金持续涌入，每次回调都有人接 | 50→100%+ |
| ⑤ 过热/分歧 | 估值极端，空头出现，波动加大 | 顶部区域 |

**系统目标：在阶段②就拍肩膀。** 不替人决策，保证人不会因为注意力带宽而错过。

## 二、第一性原理推导

### 已证伪的路径

| 路径 | 结论 | 来源 |
|------|------|------|
| 单因子机械择时 | 10 信号 + 12 参数扫描全部跑输 B&H | 2026-03-18 定论 |
| 暴力因子挖掘 | ROI 不匹配一人交易台 | Codex1 删除 2026-03-13 |
| VIX 跨资产择时 | 触发太稀疏（5 年 ~7 次），85% 持现金错过复利 | timing backtest |

### 因子层级论

所有市场信号的源头都是同一个动作：**交易者下了一笔单**。关键是我们在因果链上多远的地方观测：

```
Layer 0（原始行为）  订单簿 / Level 2         ← 散户不可及
Layer 1（直接结果）  成交量、日美元成交额       ← 最原始可及信号
Layer 2（自身对比）  RVOL（vs 历史均值）        ← 异常检测
Layer 3（跨截面）    RS 排名、PMARP            ← 二阶衍生
Layer 4（聚合层）    主题热力图、板块动量        ← 三阶衍生
```

**核心判断：成交量在因果链最底层、决策链最前端。它是最原始的，所以最早响。**

### 为什么不用期权链和社交链

市场上存在三条平行的行为链（股票/期权/信息），但信噪比差距决定了优先级：

| 链 | 测量什么 | 信噪比 | 原因 |
|----|---------|--------|------|
| **成交量** | 已完成的交易（真金白银换手） | **最高** | 最高承诺成本，伪信号少 |
| 期权 | 杠杆化的方向性赌注 | 中 | 混入大量对冲/做市/roll，非方向性操作多 |
| 社交 | 公开讨论和关注度 | 低 | 零成本发言；有价值的信息早期不会出现在公开平台 |

**结论：聚焦成交量单链，把它榨干。** 期权和社交作为可选的附加验证，不作为核心信号。

## 三、成交量的三个维度

从 RVOL 一个指标的不同观测角度，提取三个正交维度：

```
维度          │ 测量什么                       │ 对应已有组件
─────────────┼───────────────────────────────┼──────────────
① 异常强度    │ 单日 RVOL σ 值                │ ✅ RVOL 日扫描
② 时间持续性  │ 连续几天 RVOL > 2σ            │ ✅ rvol_sustained
③ 空间扩散    │ 同主题内几只同时放量            │ ❌ 缺失（本系统要建的）
```

三维组合的逻辑：

> **一只股票放了一天量 = 噪音（每天都有）**
> **一只股票连续放量 3 天 = 有人在持续建仓**
> **几只相关股票同时连续放量 = 一群人在同一个方向干活 = 趋势正在形成**

这就是**阶段②的机械化定义**。

## 四、系统架构

### 双层扫描

```
┌──────────────────────────────────────────────────┐
│  Layer A: 池内精扫（~133只，已有主题标签）           │
│  目的：已知主题在升温吗？                           │
│  信号：同主题 ≥2 只 RVOL ≥ 2σ（含持续放量）         │
│       + 涨跌方向确认                               │
│  数据源：现有 market.db                            │
│  频率：每日                                       │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────┐
│  Layer B: 池外广扫（~2500只，无主题标签）            │
│  目的：有什么我完全没注意到的东西在异动？             │
│  信号：RVOL ≥ 3σ + 涨 ≥ 3% + 不在现有池内          │
│  数据源：yfinance 免费批量下载（无状态，每日拉6个月） │
│  频率：每日，输出"新面孔"列表                       │
└──────────────────────────────────────────────────┘
```

### 信号分级

| 级别 | 条件 | 预期频率 | 阅读优先级 |
|------|------|---------|-----------|
| 🔴 强信号 | 同主题 ≥2 只 RVOL ≥ 2σ + 持续 ≥2 天 + 上涨 | 每周 0-2 次 | 立刻看 |
| 🟡 观察信号 | 单只 RVOL ≥ 3σ + 涨 ≥ 5% 或持续放量 ≥ 3 天 | 每天 3-8 只 | 扫一眼 |
| 🔵 新面孔 | 池外股票 RVOL ≥ 3σ + 涨 ≥ 3% | 每天 5-15 只 | 留意重复出现的 |

### 推送格式（晨报集成）

```
━━━ 热点追踪 ━━━

🔴 Memory 主题异动（连续第 2 天）
   MU   | RVOL 4.2σ (持续3天) | +5.3% | 财报 3 天前
   WDC  | RVOL 3.1σ (持续2天) | +3.8% |

🟡 单只观察
   COIN | RVOL 5.1σ | +8.2% | crypto 主题（无联动）

🔵 池外新面孔
   LUNR | RVOL 6.3σ | +12% | 不在池内（space?）
   RKLB | RVOL 3.8σ | +7%  | 不在池内（space?）
   ↑ LUNR 连续出现 3 天
```

## 五、需要建的东西

### 5.1 主题字典 (Theme Dictionary)

- 10-20 个主题，手工定义 ticker → theme 映射
- 现有 `THEME_KEYWORDS_SEED` 可作为起点（25 个主题），精简合并后维护 ticker 列表
- 初期只覆盖池内 ~133 只，后续按需扩展
- 存储位置：`config/settings.py` 或独立 JSON

### 5.2 主题联动检测 (Theme Co-movement Detector)

- 输入：每日 RVOL + 涨跌幅 + rvol_sustained 数据
- 逻辑：按主题分组 → 统计同时触发的数量 → 生成信号级别
- 输出：分级信号列表

### 5.3 池外广扫 (Broad Market Scanner)

- FMP screener → 全量 ticker 列表（市值 > $5B，~2500 只）
- yfinance 批量下载 6 个月数据（~90 秒，免费，无状态）
- 计算 RVOL + 涨跌过滤
- 排除现有池内 ticker → 输出"新面孔"
- 跟踪连续出现天数（简单 JSON 文件即可）

### 5.4 晨报集成

- 将热点追踪信号整合到现有晨报推送流程

## 六、不做什么

- **不做交易信号** — 系统是注意力分配器，不是买卖触发器
- **不做自动聚类** — 已证明不可用，主题靠人工定义
- **不引入期权链/社交链作为核心信号** — 信噪比不够，可作为附加上下文
- **不建历史数据库给池外广扫** — yfinance 无状态拉取，避免维护负担

---

## 七、定位

本系统是**注意力代理**——机械化地完成"每天扫一遍所有股票，看有没有什么不寻常的"这件人类做不到的事。它不需要预测未来，只需要可靠地检测异常并分级推送。最终决策权始终在人。

这是 CIO 痛点 #3（"新主题出现早期总是错过"）的系统化解法。
