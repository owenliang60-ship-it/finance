## 各层定义

### 第一层：数据层（地基）

日频自动更新的原始数据，为上层分析提供输入。

| 数据类型 | 来源 | 存储 | 更新频率 |
|---------|------|------|---------|
| 量价 | FMP API | market.db:daily_price | 日频 (云端 cron) |
| 基本面 | FMP API | market.db:income/BS/CF/ratios/metrics | 周频 (云端 cron) |
| 期权 IV | MarketData.app | market.db:iv_daily | 日频 (云端 cron) |
| 宏观 | FRED API | macro_snapshot.json (4h/12h TTL) | 准实时 |
| 前瞻预期 | yfinance | market.db:forward_estimates/metadata | 周频 (云端 cron) |
| 社交情感 | Adanos (Reddit + X) | market.db:social_sentiment | 日频 (云端 cron 06:55) |
| 新闻 | FMP API | 按需获取 | 按需 |

### 第二层：分析层（三维度）

对原始数据的深加工，产出每只股票在三个维度的分析结论。

**基本面分析（季频）**
- 驱动方式：Knowledge 层（6 lens + 辩论协议 + memo 模板）
- 产出：深度研究报告 + 基本面评分
- 更新触发：财报发布、重大事件、分析过期
- 当前状态：成熟（15 agent deep pipeline）

**技术分析（日频）**
- 驱动方式：经因子研究验证过的技术指标
- 产出：技术面信号（趋势、动量、超买超卖、量能异常）
- 更新触发：每日随数据更新自动计算
- 当前状态：薄弱（仅 PMARP + RVOL），待补齐

**情绪分析（日频）**
- 驱动方式：经因子研究验证过的情绪因子
- 产出：市场情绪信号（热度、恐惧/贪婪、社交趋势）
- 更新触发：每日随数据更新自动计算
- 当前状态：雏形（Attention Engine），待补齐数据源和分析框架

### 第三层：策略层（信号合成）

汇总三个分析维度，在任意时间点对任意股票产出一个明确的建议仓位与动作。

- 核心框架：OPRMS（将从纯基本面演化为三维合成）
- 核心难题：三个维度"打架"时如何调和（概率思维 + 完整交易系统）
- 输出格式：单票建议仓位 + 动作（建仓/加仓/持有/减仓/清仓）+ 置信度
- 当前状态：OPRMS 存在但仅反映基本面维度

### 第四层：CIO 层（组合构建）

整合所有单票策略建议为一个可执行的投资组合。

- 核心能力：N 只股票的独立建议 → 一个合理的组合
- 输入：所有策略层建议 + 宏观/Regime 环境 + 当前持仓
- 行为纠偏：作为认知偏差防火墙（损失厌恶、确认偏差、禀赋效应等）
- 风格：集中型（5-15 只，单票 15-25%），参照传奇个人投资者
- 当前状态：研究计划已立项（`docs/plans/2026-03-03-cio-layer-research-plan.md`）

### 侧翼：宏观/Regime

不属于个股分析，而是定义市场环境，供 CIO 层参考。

- Regime 分类：CRISIS / RISK_OFF / RISK_ON / NEUTRAL
- 跨资产信号：5 个检测器（carry unwind, credit stress, liquidity drain, reflation, risk rally）
- 投资主题/热点：Theme Engine（量价动量 + 注意力量化）

### 离线 R&D：回测引擎 + 因子研究

不在生产管道内，是发现和验证有效因子的实验室。

- 回测引擎：策略级验证（"这个策略赚钱吗？"）
- 因子研究：因子级验证（"这个因子有预测力吗？"）
- 产出：经验证的因子 → 纳入技术分析或情绪分析层

## 各层完成度

| 层 | 完成度 | 关键缺口 |
|---|---|---|
| 数据层 | ~95% | 期权链数据有限（暂缓） |
| 分析层 — 基本面 | ~90% | 基本齐备 |
| 分析层 — 技术 | ~20% | 需更多经验证的指标 |
| 分析层 — 情绪 | ~15% | 数据已有，缺因子定义+分析框架 |
| 策略层 | ~15% | OPRMS 需三维化 |
| CIO 层 | ~5% | 无实现 |
| 离线 R&D | ~60% | 需更多因子挖掘 |
| 宏观/Regime | ~70% | 热点/主题引擎待加强 |

---

## 建设路径

自下而上，逐层夯实：

1. **补齐分析层**：技术分析指标扩充 + 情绪因子定义与验证（数据层已就绪）
2. **演化 OPRMS**：从纯基本面评级 → 三维合成策略输出
3. **构建 CIO 层**：组合构建 + 行为纠偏

每一步都不跳跃——分析层三条腿站稳了，策略层自然有东西可以合成；策略层有了可靠的单票建议，CIO 层才有素材做组合。
