**X 路径核心**:系统不生成"新建议",而是**把 Boss 在平静时刻写下的结构化契约,在风暴中回显给 Boss 看**。系统不是决策大脑,是契约执行复读机。

**CIO-A 的新物种定义(永久性)**:

>
> 它把 Boss 在平静时刻写下的结构化决策契约(thesis + kill condition + 触发规则)**打包存储**,然后把**当前状态对所有契约做匹配**,风暴中清晰地回显给 Boss:"你过去的自己在这种情况下说了什么"。
>
> Boss 的心智任务从"**在情绪中做新决策**"降维成"**识别契约是否被触发**" —— 这是情绪中人脑唯一还能可靠做的事,也正是顶级运动员在世界杯决赛场上做的事。

## D1 动机维度 — 最终结论

### 核心命题

CIO-A 是 Boss 的**决策锚点系统 + Dashboard Platform 接入者**:

- **业务目的**:把过去理性状态下的思考持久化、可视化、风暴可读,让"坚守既定策略"**从意志力任务降维为信息检索任务**
- **使用定位**:只回答"现状 + 过去的你怎么想的",**不回答"现在该怎么动"**。Thesis 修订权由客观 regime 信号授权,不由主观情绪触发
- **设计哲学**:**Harm Reduction** — 不对抗 Boss 风暴中反复查看的行为,而是**截流**这个行为,让 Dashboard 成为券商 app 的更好替代品
- **架构定位**:CIO-A 是 CIO 层共享基建(Dashboard Platform)的**第一个 view provider**
- **目标函数**:**20% 风暴日的可用性 + 注意力争夺力 + 长期可扩展性**(不优化 80% 平静日)

### 五层辐射影响(X 路径确立后,不可动摇)

| # | 层 | 影响 |
|---|---|---|
| 1 | **数据层** | 新增 `trading_rules` 结构化契约表;thesis 从自由文本 → 结构化契约 + 描述文本双轨制 |
| 2 | **计算层** | 新增**契约匹配引擎** — 定期扫描所有 OPEN 持仓,把当前状态对所有契约做匹配,输出 triggered/pending/far 三态 |
| 3 | **DSL 层** | 新增**契约语言**(类似 DSL),让 Boss 能写"PMARP<30 且 kill 未触发 → 加仓至 25%"这样的规则 |
| 4 | **呈现层** | Dashboard 首屏 C 位必须是"契约触发面板" |
| 5 | **行为工作流** | `/trade` skill 必须支持"写契约"步骤;**每次买入行为被升级为"写下未来的剧本"** |

### 关键架构约束

- **信息呈现 >>> 信息生成**(风暴中读不懂 = 零价值)
- **Web Dashboard 主入口**(Telegram 降级为通知层)
- **必须消费 Regime 信号**作为 thesis 修订的客观授权
- **严禁越界**:不给加减仓建议,不做交易预演,不做探索式研究

## D2 用户维度 — 最终结论

### 4 种真实用户人格(Boss 确认)

| 代号 | 场景 | 认知状态 | 关心的事 | 由谁服务 |
|---|---|---|---|---|
| **A** 晨间复盘者 | 周末/工作日早晨,咖啡在手 | 冷静、有耐心 | 全局概览、trend、kill 审视 | **Dashboard** |
| **B** 盘中被动感知者 | 工作日白天,瞥一眼 | 半分神 | 顶层 3-5 个数字、告警 | **Telegram 推送** |
| **C** 风暴救火队员 | 急跌/急涨/黑天鹅日 | 情绪激动、反复刷 | thesis 立即可见、kill 显著 | **Dashboard** |
| **F** Regime 复审者 | 市场换挡被系统提示时 | 半冷静半警觉 | thesis 是否站得住 | **专门报告**(独立交付物) |

### 排除的人格(Boss 明确排除)

- **D** 交易决策核对 → 归 `/trade` skill,Dashboard 不重复
- **E** 周末研究者(交互式情景压测) → 排除,Dashboard 只展示**预设**情景(-5% / -10% / -20% 三档)

### Dashboard 双峰挑战(留给 D4 产品形式细节)

A(冷静探索式) 和 C(激动扫描式) 是对立认知状态,同一 Dashboard 要服务两者。三种可能解法,**D4 会最终决定**:
- 思路 1:**倒金字塔分层**(首屏 C 友好,下滚 A 友好) — 我的倾向
- 思路 2:**双模式显式切换**
- 思路 3:**访问频率自适应**

### 宪法级约束(不可动摇)

**"Dashboard 是事后管理,不涉及交易本身"** — 来自 Boss 原话。Dashboard 严格服务于"持仓生命周期的中间阶段":

```
交易前研究         交易决策时刻         ★ Dashboard 领地 ★         Regime 换挡
   ↓                 ↓                       ↓                      ↓
深度分析 pipeline → /trade skill →  事后管理 + 锚点 + 纪律捍卫  → 强制复审 → 新一轮深度分析
```

**Dashboard 硬边界**:
- ❌ 交易预演 / 交互式研究 / 任何写入动作 / 下单入口
- ✅ 现状呈现 + 历史演变 + thesis/契约回显 + 预设情景压测 + regime 事件响应

**Dashboard 是完全只读的**。

## 下次启动:第一个问题

> 2026 年某个周六早晨 9 点,你泡好咖啡坐在书房,打开 Dashboard。这是一个平静的早晨,市场没有发生任何特殊事件,你就是想"看看我的组合最近怎么样"。你有大约 **20-30 分钟**专门给这件事。

请描述从这一刻到你合上屏幕为止的**完整动作-响应序列**。具体到:

- **第 1 分钟**:首屏你在看什么?是看"本周发生了什么"?还是"目前的状态健康度"?还是"最近哪只股票动静大"?
- **第 5 分钟**:你在想什么?是在"学习市场"?是在"审视我自己的决策质量"?还是在"找异常"?
- **第 10 分钟**:你会下钻到哪个具体持仓?是最近跌得多的那只?还是你最担心 thesis 的那只?还是你最不熟悉的那只?
- **第 15-20 分钟**:你会做什么?是回顾某只持仓的 thesis + 契约?是看组合整体的演变?是看"如果再跌 10% 会怎样"的预设情景?还是做别的事?
- **带走什么**:周六早晨的复盘结束时,你希望带走一个什么样的"结论"?

**顺便碰一下的隐含问题**:
1. 你会不会有"周对比"的需求?
2. 你会不会想看"每只股票 thesis 距离 kill condition 还有多远"的温度计?
3. 你会不会想看"这段时间发生了哪些契约触发事件"的时间线?
4. 晨间复盘是更像"学习/反思"(内省),还是更像"侦察/扫雷"(外查)?

**一个元问题**:你现在每周/每月有没有正式的复盘习惯?Dashboard 是要**创造一个新习惯**,还是要**服务一个已经存在的习惯**?
